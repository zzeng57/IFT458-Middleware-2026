Book Exchange Platform — React Native + Express API

Project Overview

This project got two parts. The backend is a Express.js API server that handle user authentication (signup/login) and book CRUD operations with MongoDB. The mobile folder is a React Native app built with Expo that connect to the backend API. The mobile app got all the same features like the web version (login, register, book exchange, logout) but it run on your phone or in a emulator.

Backend Setup

The backend is a Express.js server that provide the REST API. It connect to MongoDB for storing users and books. You need to start this first before the mobile app.

1 Open Terminal and Navigate

Open your terminal (Command Prompt, PowerShell, or Mac Terminal) and go to the backend folder:

cd "Module 4 Activity 1 ReactNative/backend"

2 Install Dependencies

Install all the Node.js packages that the backend need:

npm install

This will read package.json and download all the required libraries like express, mongoose, jsonwebtoken, bcryptjs, cors, etc.

3 Check config.env

Make sure the config.env file got your MongoDB connection string and JWT secret:

DATABASE=mongodb+srv://your-connection-string
JWT_SECRET=your-secret-key
JWT_EXPIRES_IN=90d
JWT_COOKIE_EXPIRES_IN=90
PORT=4000
Important: If you dont have a MongoDB Atlas account yet, go to mongodb.com/cloud/atlas and create a free cluster first.

4 Start the Server

npm start
If it work you should see:
App running on port 4000...
DB connection successful!

The API is now live at http://localhost:4000

Keep this terminal open! The backend must stay running while you use the mobile app. If you close the terminal the API stop and the mobile app cant connect.

Mobile App Setup

The mobile app is built with React Native and Expo. Expo make it easy to run React Native apps without needing Android Studio or Xcode.

1 Prerequisites

Make sure you got these installed on your computer:

ToolCheck CommandInstall
Node.js v16+ node --version nodejs.org
npm npm --version Come with Node.js
Expo CLI npx expo --version npm install -g expo-cli

2 Find Your IP Address

Your phone cant use localhost because that mean the phone itself. You need your computer IP address on the WiFi network.

Mac▶ 
ipconfig getifaddr en0

This give you something like 192.168.1.45

Windows▶ 
ipconfig

Look for "IPv4 Address" under your WiFi adapter. It look like 192.168.1.xxx

Linux▶ 
hostname -I

Take the first IP address it show you.

3 Update the API URL

Open the file mobile/utils/api.js and find the PHONE_API_BASE variable. Change the IP to YOUR computer IP:

// Change this line to your computer IP address
const PHONE_API_BASE = 'http://192.168.1.YOUR_IP:4000/api/v1';
This is the #1 most common mistake! If you skip this step the app will show "Network Error" when you try to login or register from your phone.

4 Install Mobile Dependencies

Open a new terminal window (keep the backend running in the other one) and navigate to mobile folder:

cd "Module 4 Activity 1 ReactNative/mobile"
npm install

5 Start Expo

npx expo start
You should see a QR code in your terminal! This QR code is what you scan with your phone to open the app.

Device Options

You got 3 ways to run the mobile app. Pick the one that work best for you.

Option 1: Web Browser (Easiest)

After running npx expo start, press w in the terminal to open the app in your browser.

Simulate iPhone in Chrome:

1 Open Chrome DevTools (F12 or Ctrl+Shift+I)

2 Click the device toggle button (looks like a phone/tablet icon)

3 Select "iPhone 14 Pro" or any iPhone from the dropdown

4 Click the 3-dot menu, pick "Show device frame" for the iPhone look

Tip: This is the fastest way to test. No phone needed at all. The app look and work the same as on a real phone.

Option 2: Real Phone with Expo Go

This is the most realistic way to test your app. It run on your actual phone as a native app.

Setup steps:

1 Download Expo Go from App Store (iPhone) or Google Play (Android)

2 Make sure your phone and computer are on the same WiFi network

3 Run npx expo start in the terminal

4 Scan the QR code with your phone camera (iPhone) or Expo Go app (Android)

Your phone and computer MUST be on the same WiFi! If they on different networks the app will not connect. Also make sure you updated the IP address in api.js.

Option 3: Emulator (Advanced)

If you want a virtual phone on your computer screen, you can use a emulator. This need extra software installed.

Android Emulator (Android Studio)▶

1 Install Android Studio

2 Open Android Studio > Tools > Device Manager

3 Create a Virtual Device (pick Pixel 7 or any phone)

4 Start the emulator first, then press a in the Expo terminal

Android Studio is a big download (~2GB). Only use this if you really want a emulator.
iOS Simulator (Mac Only)▶

1 Install Xcode from the Mac App Store

2 Open Xcode > Preferences > Components > install a iOS Simulator

3 Press i in the Expo terminal to launch iOS simulator

Xcode is only available on Mac and is a very big download (~12GB).

Testing the App

Follow these test to make sure everything is connected and working good.

Your testing progress (click to mark done):

Register → Login → View Books → Add Book → Logout

1 Register a New User

Open the app and tap Register. Fill in name, email, password, and confirm password. Tap the signup button.

Expected result: You see the "Authorization Successful" screen with your name and a JWT token. This mean the backend and mobile are connected good.

2 Login

Go to the Login screen. Enter the email and password you just created. Tap login.

Expected result: You see the success screen again with your user info.

3 View Book Exchange

After logging in, tap Books in the bottom navigation. This screen fetch all books from the API.

Expected result: You see a list of books (or a "No books available" message if the database is empty). If you see a error, your JWT token might be invalid - try logging in again.

4 Add a Book

On the Book Exchange screen, tap Add a New Book. Fill in the title, author, description, and pick a exchange type (Borrow or Trade).

Expected result: The book get added and you go back to the book list where you can see your new book.

5 Logout

Tap Logout in the bottom navigation. Confirm by tapping "Yes, Logout".

Expected result: You go back to the home screen. The bottom nav now show Login and Register instead of Books and Logout. Your session is cleared.

How Authentication Work

When you login or signup, the server send back a JWT (JSON Web Token). The mobile app save this token in the phone storage. Every time you want to access a protected route (like books), the app send the token in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

The backend middleware check this token. If it valid, you get access. If not, you get a 401 error.

Troubleshooting

Here are the most common problem students run into and how to fix them.
"Network Error" when trying to login/register▶

Cause: The mobile app cant reach the backend server.

Fix:

1. Make sure the backend server is running (npm start in backend folder)

2. Check that you updated the IP address in mobile/utils/api.js

3. Make sure phone and computer are on the same WiFi

4. Try using http:// not https:// in the API URL

"DB connection failed" error in terminal▶

Cause: MongoDB connection string is wrong or MongoDB Atlas is not set up.

Fix:

1. Check config.env file - make sure the DATABASE url is correct

2. Make sure you replaced <password> with your actual MongoDB password

3. Go to MongoDB Atlas and make sure your IP is in the whitelist (or add 0.0.0.0/0 for allow all)

Blank white screen in browser▶

Cause: Usually a JavaScript error that stop the app from rendering.

Fix:

1. Open browser DevTools (F12) and check the Console tab for red errors

2. Make sure package.json has "main": "node_modules/expo/AppEntry.js"

3. Try deleting node_modules and running npm install again

4. Clear browser cache and reload

EMFILE: too many open files▶

Cause: Metro bundler is trying to watch too many files (including the backend folder).

Fix: Make sure metro.config.js exist in the mobile folder and it exclude the backend folder from watching. Also make sure .watchmanconfig file exist.

App look like a small "postage stamp" in browser▶

Cause: Expo web by default show the app in a narrow mobile-width viewport.

Fix: The App.js file already got CSS injection code that fix this. If it still look small, try using Chrome DevTools device toolbar to simulate a iPhone screen instead.

Expo Go cant find the app / QR code not working▶

Fix:

1. Make sure both device are on the same WiFi network

2. Try switching Expo to tunnel mode: npx expo start --tunnel

3. This install @expo/ngrok and create a tunnel that work across networks

4. On Android, scan the QR code from inside the Expo Go app (not the camera)

"401 Unauthorized" on Book Exchange page▶

Cause: Your JWT token is expired or not being sent correctly.

Fix:

1. Try logging out and logging back in

2. Check that api.js is sending the Authorization header with the saved token

3. Make sure the backend authMiddleware.js accept Bearer tokens (not just cookies)

react-native version mismatch warning▶

Cause: The installed react-native version dont match what Expo SDK 50 expect.

Fix: Make sure package.json got "react-native": "0.73.6" (not 0.73.4). Then delete node_modules and run npm install again.
